# 防合并分支

# 业务场景

当我们预发环境不够用时（比如只有一两个预发环境），但是同时有多个人需要上预发测试，通常我们的做法是将要测试的分支合并到一个测试主分支，例如（stage分支），这个分支通常会有不同版本的代码同事测试，为避免未上线的测试代码合并到主分支，我们需要限制stage合并到开发分支

# 思路一

首先我们在预发环境增加一个.stage文件，提交上去。

使用husky的钩子函数（钩子函数就不一一讲了）pre-commit，在提交（git commit）的时候会触发这个钩子函数，写一个node检测文件的脚本，命名为：checkEnv.js

const fs = require("fs")

const path=require('path');
const defPath=path.join(__dirname,'../../')
const childProcess = require('child_process')
const branch = childProcess.execSync('git rev-parse --abbrev-ref HEAD').toString().replace(/\s+/, '')
const files = ['.stage','.stage1'];
if(!files.includes(`.${branch}`)) {
    for (let i = 0, len = files.length; i < len; i++) {
        const item = files[i];
        // 检查文件是否存在于当前目录中。
        try {
            fs.accessSync(defPath + item, fs.constants.F_OK);
            console.warn("\x1B[31m 存在"+ item.replace('.','') +"代码，请还原！" + "\x1B[0m")
            process.exit(1);
        } catch (err) {
        }
    }
}
