# Vue Route 参数传递应避免的坑

# route参数传递引起的BUG

在vue-router跳转时候，应避免传递的参数带有非String类型的值
比如三个页面，A->B->C,A到B的时候，页面传递参数flag=true，从C页面返回到B的时候，query里面的参数flag会变成'true'， 如果参数是int类型，比如传递参数num=1，从C页面返回，query里面参数num会变成'1'，如果有计算会导致计算错误。

WARNING

不仅回退会导致这个问题，刷新也会

# 从源码找原因

History.prototype.updateRoute = function updateRoute (route) {
  var prev = this.current;
  this.current = route;
  this.cb && this.cb(route);
  this.router.afterHooks.forEach(function (hook) {
    hook && hook(route, prev);
  });
};

跳转的时候使用push方法，路由其实是使用新路由赋值给当前路由，所以参数route里面的值到下一个页面会保留原有的数据类型。 页面回退和刷新属于浏览器行为，route获取到的query其实是浏览器url里面的参数，所以全部为String类型，可以参考createRoute实现

function createRoute (
  record,
  location,
  redirectedFrom,
  router
) {
  var stringifyQuery$$1 = router && router.options.stringifyQuery;

  var query = location.query || {};
  try {
    query = clone(query);
  } catch (e) {}

  var route = {
    name: location.name || (record && record.name),
    meta: (record && record.meta) || {},
    path: location.path || '/',
    hash: location.hash || '',
    query: query,
    params: location.params || {},
    fullPath: getFullPath(location, stringifyQuery$$1),
    matched: record ? formatMatch(record) : []
  };
  if (redirectedFrom) {
    route.redirectedFrom = getFullPath(redirectedFrom, stringifyQuery$$1);
  }
  return Object.freeze(route)
}
